## Phase Goal

**As an** operator, **I want to** confirm that incremental SSE streams pass through the BFF and fronting proxy unbuffered, **so that** later streaming features (Flow live run, Chat) are guaranteed to work before any streaming UI is built.

<objective>
Prove the BFF-03 keystone: the thinnest possible end-to-end SSE slice through the real fronting nginx.

Purpose: De-risk the entire streaming architecture before any streaming UI is written. A misbehaving proxy silently batches streams — this plan makes the failure mode loud and detectable before Phase 3 investment.

Output: Go BFF skeleton (config loader + `/api/stream/test` synthetic SSE handler + `/healthz`), nginx config with correct SSE location directives, docker-compose.yml for the compose stack, and a `scripts/sse-proof.sh` that fails if nginx is buffering. Go unit tests for the SSE handler. BFF-03 gate is provable via `go test` AND `curl -N` both directly (:8090) and through nginx (:80 via docker-compose).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → nginx | Untrusted: any HTTP request from any origin; nginx is the first hop |
| nginx → BFF `/api/stream/test` | Internal: nginx proxies to BFF on internal Docker network; BFF has no upstream auth on this synthetic endpoint |
| BFF → config file | BFF process reads YAML config from filesystem; secret field `flowd_token` / `operator_token` are present but empty in dev |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | `config/config.dev.yaml` flowd_token field | mitigate | `flowd_token` and `operator_token` fields are empty in the committed dev config; production values are injected via env-var override or a non-committed config file; grep gate in acceptance criteria confirms no hardcoded token value |
| T-01-02 | Tampering | nginx SSE location block ordering | mitigate [BLOCKING] | SSE location `~* ^/api/.*stream` MUST precede general `/api/` block; wrong ordering silently removes `proxy_buffering off` from SSE routes; acceptance criteria verify line ordering |
| T-01-03 | Denial of Service | `/api/stream/test` synthetic SSE endpoint | accept | Endpoint is for the BFF-03 transport proof only; no auth required by design (transport proof, not auth proof); in production, the endpoint is gated by the app-layer operator token middleware which is added in Plan 03; low risk in dev where the endpoint is used |
| T-01-04 | Tampering | npm/pip/cargo installs | mitigate | Package Legitimacy Audit completed in RESEARCH.md; all packages approved (see audit table); `gopkg.in/yaml.v3` is a well-known stdlib-adjacent YAML library |
| T-01-SC | Tampering | go get / npm install package installs | mitigate | slopcheck + Package Legitimacy Audit table in RESEARCH.md covers all packages; no [ASSUMED] or [SUS] packages in this plan's installs |
