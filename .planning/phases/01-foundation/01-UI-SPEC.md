# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope of this contract:** the **app shell + cross-cutting design system + reusable primitives** only. Per-service feature screens (Memory in Phase 2, Flow in Phase 3, Chat in Phase 4) consume these tokens and primitives but are NOT specified here. The nav targets Memory / Flow / Chat are **placeholders** in this phase.
>
> **Aesthetic mandate (operator tool):** legibility and state-clarity over marketing polish. The cardinal sin is **silent failure** — every contract below exists to make state and outcome unmistakable. Calm, dense, dark-first; nothing decorative competes with status.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (CLI 3.x) — LOCKED by research/STACK.md; not re-litigated here |
| Preset | not applicable — initialized during Phase 1 execution with the tokens below as the baseline theme (no marketplace preset) |
| Component library | Radix primitives (via shadcn/ui) on Tailwind v4 |
| Icon library | lucide-react (shadcn default) — `size-4` (16px) default, `size-3.5` (14px) for status dots/inline |
| Font (UI) | system UI sans stack: `ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` |
| Font (mono) | `ui-monospace, "JetBrains Mono", "SF Mono", "Cascadia Code", Menlo, Consolas, monospace` — used for **all ids, raw JSON, env/endpoint, event payloads** |

**Theme mode:** Single **dark-first operator theme** (one fixed theme, no toggle). A switchable light/dark mode is a v2 deferred item (REQUIREMENTS.md); a dark *default* surface is the operator aesthetic, not the deferred feature. Tokens below are the dark theme. Define them via shadcn CSS variables (`--background`, `--foreground`, `--card`, `--border`, `--primary`, `--destructive`, plus custom `--status-*`) so a light theme can be added later without re-tokening.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, status-dot-to-text gap, inline chip padding |
| sm | 8px | Compact element spacing, copy-button hit padding, toast internal gap |
| md | 16px | Default element spacing, shell content padding, card padding |
| lg | 24px | Section padding, gap between shell regions |
| xl | 32px | Layout gaps, empty-state vertical rhythm |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing / empty-state top offset |

**Density:** This is a dense operator tool, not a marketing site. Default to `md` (16px) for content padding; use `sm` (8px) inside the operator-context bar, nav rows, and toolbars to keep chrome compact.

Exceptions:
- **Interactive hit targets** — icon-only buttons (copy-id, copy-JSON, JSON collapse toggle) render visual size at 16–20px but MUST carry a **minimum 32px clickable target** (padding fills the gap). Keep visual density without sacrificing clickability.
- **Status dots** — 8px diameter (`sm`), a deliberate sub-grid size for the up/down/degraded indicator; not a spacing token.
- **Shell chrome heights** — top bar 48px (`2xl`), left nav width 56px collapsed / 224px expanded. Fixed, not on the 4-scale's typical content rhythm but both multiples of 4.

## Typography

Exactly 4 sizes, exactly 2 weights (regular 400, semibold 600). No italic, no light, no bold-700.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 20px | 600 | 1.3 | Console/section title in the shell header, empty-state heading |
| Heading | 16px | 600 | 1.4 | Card titles, panel headers, dialog titles |
| Body | 14px | 400 | 1.5 | Default operator text, descriptions, nav labels, empty/error body copy |
| Label | 12px | 600 | 1.4 | Operator-context keys (TENANT, USER), status captions, last-checked timestamps, table column headers — uppercase with `0.04em` letter-spacing |

**Monospace usage** (same size scale, mono family): all **ids, env/endpoint strings, raw JSON, and event payloads** render in the mono font at Body (14px) or Label (12px) size. Ids are mono so they are visually distinguishable from prose and unambiguous to copy/compare.

## Color

Dark-first operator palette. Values are the locked dark theme. Calm desaturated neutrals so that **status color is the only thing that pops** — the eye must go straight to a red/amber/green state.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0B0E14` (near-black slate) | App background, main content surface |
| Secondary (30%) | `#151A23` (elevated slate) — borders `#232A36` | Cards, left nav, operator-context bar, top bar, JSON viewer surface |
| Accent (10%) | `#3B82F6` (blue 500) | See reserved-for list below |
| Destructive | `#EF4444` (red 500) | Destructive-action buttons + destructive confirmation dialogs only |

Foreground: primary text `#E6EAF0`, muted text `#8A93A3` (timestamps, secondary labels), on a `#0B0E14`/`#151A23` surface — all body/label combos meet WCAG AA (≥4.5:1).

**Accent (blue) reserved for — explicit list, never "all interactive elements":**
- The **active nav item** (selected console in the left nav).
- The **primary button** in a view (e.g. "Set context" / "Save context" in the operator-context editor — the single primary action per surface).
- **Focus rings** (keyboard focus outline) — accessibility, applies app-wide.
- Inline **links / copyable-id hover affordance**.

Everything else (secondary buttons, nav inactive items, borders, icons) stays neutral. No blue on cards, panels, or decorative surfaces.

### Status colors (operator-tool semantic layer — REQUIRED, beyond the 60/30/10 split)

These are functional state signals, not part of the accent budget. They are the whole point of the tool. Each pairs a **dot color + text + icon** so status never relies on color alone (color-blind safe):

| State | Color | Token | Used by | Icon (lucide) |
|-------|-------|-------|---------|---------------|
| Up / healthy / success | `#22C55E` (green 500) | `--status-up` | health dots (up), success toast | `check-circle` |
| Degraded / warning / partial | `#F59E0B` (amber 500) | `--status-degraded` | health dots (degraded), partial-data state | `alert-triangle` |
| Down / error | `#EF4444` (red 500) | `--status-down` | health dots (down), error state, error toast | `x-circle` / `alert-circle` |
| Unknown / checking / loading | `#8A93A3` (slate, muted) | `--status-unknown` | health dot before first poll, loading state | `loader` (spin) / `circle-dashed` |

`--status-down` and `Destructive` share the same red. Acceptable: both mean "danger/failure"; context (a dot vs. a button) disambiguates.

## Copywriting Contract

Tone: **terse, precise, operator-grade.** No marketing voice, no apologies, no "Oops!". State the fact, surface the real upstream detail, point to the next action. Sentence case. Errors MUST carry the actual upstream status + message (BFF-04 passes them through) — never a generic "Something went wrong."

| Element | Copy |
|---------|------|
| Primary CTA (operator context) | **Set context** (first time, when unset) / **Save context** (editing existing) |
| Primary CTA (shell nav placeholders) | Each placeholder body: **"{Memory/Flow/Chat} console — arrives in Phase {N}."** (no fake CTA; placeholder is explicitly labeled) |
| Empty state heading | **No operator context set** |
| Empty state body | **Set a tenant id and user id to reach the backends. Project and session are optional.** + primary button "Set context" |
| Error state | **{HTTP status} from {service} — {upstream message}.** Below: muted "Retry" button + "View raw JSON" disclosure. Example: `503 from memory-gateway — upstream unavailable.` Never collapse the status/message into a generic string. |
| Loading state | **Loading…** (with `loader` spin icon) — distinct from empty and from error; never a bare blank panel. |
| Partial state | **Showing partial data — {what's missing/why}.** Amber `alert-triangle`. Example: `Showing partial data — health check for flowd timed out.` |
| Toast — success | Terse confirmation in past tense: `Context saved.` (≤ ~40 chars; auto-dismiss ~4s) |
| Toast — failure | `{action} failed — {HTTP status}: {upstream message}.` Persists until dismissed; includes a "Copy error" affordance. Example: `Save context failed — 400: tenant_id required.` |
| Destructive confirmation | None in Phase 1. Phase 1 ships **no destructive actions** (no delete/disable — those arrive in Phase 2 Memory). The destructive dialog *pattern* is defined here for downstream reuse: title `{Verb} {resource}?`, body states the irreversible consequence, confirm button uses `Destructive` red and repeats the verb (e.g. "Delete"), cancel is the default/neutral button and is focused by default. |

### The five-state pattern (cross-cutting contract — every later view MUST implement)

Phase 1 defines this as a reusable primitive; Phases 2–5 enforce it per view. The five states are **visually distinct** — a blank panel is a contract violation:

| State | Visual | When |
|-------|--------|------|
| **loading** | `loader` spin + "Loading…" centered, muted | request in flight, no data yet |
| **empty** | icon + heading + body + (optional) primary CTA, centered | request succeeded, zero items |
| **error** | red `alert-circle` + `{status} from {service} — {message}` + Retry + raw-JSON disclosure | request failed (upstream non-2xx or transport error) |
| **partial** | amber `alert-triangle` + partial-data banner above whatever data did load | some data loaded, some failed/degraded |
| **ready** | the actual content | request succeeded with data |

(Phase 5 adds the SSE-specific **disconnected/reconnecting** state on top of these for stream views; out of scope for Phase 1, which has no stream UI yet — but the state machine is designed to extend, not be rewritten.)

### Cross-cutting primitive contracts (built once in Phase 1, reused everywhere)

| Primitive | Contract |
|-----------|----------|
| **Operator-context bar** | Always visible in the shell. Shows `TENANT`/`USER` (and `PROJECT`/`SESSION` when set) as mono values with copyable-id affordance; a clear **"unset"** treatment (muted + "not set") when empty. Persists to localStorage across reloads. Editing opens an inline editor / popover with the "Set/Save context" primary CTA. When tenant or user is unset, the bar reads as a warning (amber) because memory is unusable without it. |
| **Active environment / endpoint indicator** | Always visible in the top bar, mono, prominent. Shows the env/endpoint the BFF targets (read-only in v1 — no switcher). Treated as load-bearing: an operator must never be unsure which environment they are acting against. |
| **Per-service health dots** | One dot per service (memory-gateway, flowd, chat) in the shell. 8px dot + status color + icon + `last-checked` Label-size timestamp on hover/adjacent. States: up / degraded / down / unknown (pre-first-poll). Phase 1 ships the **dot + placeholder/wireframe**; live polling lands in Phase 5 (SHELL-02), but the visual contract and `unknown` initial state are fixed here. |
| **Raw-JSON viewer** | Collapsible, syntax-highlighted (mono font, muted punctuation, neutral keys, restrained value coloring — no rainbow), **copy-to-clipboard** button (copies full payload, toasts "Copied"). Collapsed by default in detail/error contexts; one component reused for memory items, run events, chat steps, error bodies. |
| **Copyable id** | Any resource id (tenant/user/run/session/memory) renders mono with a hover-revealed `copy` icon (lucide `copy`); click copies and flips to `check` for ~1s + toasts "Copied". Min 32px hit target. |
| **Toast** | Single app-wide system (sonner). Success = green `check-circle`, auto-dismiss ~4s. Error = red, **persists** until dismissed, carries upstream `{status}: {message}` + "Copy error". Bottom-right. One toast per action; no stacking spam. |
