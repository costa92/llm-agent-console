## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator click → destructive mutation | Delete/disable cross a confirmation boundary (operator-error guard); weight matched to reversibility |
| Cached version → lifecycle expected_version | OCC gate: the cached version is sent as expected_version; a stale value is a recoverable conflict |
| Lifecycle response {flag,version} → cache | Echoed flag+version merged into cache; delete splices — no optimistic flip-then-revert |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02C2-01 | Tampering (lost update / OCC) | stale expected_version on lifecycle calls | mitigate [BLOCKING] | Every pin/unpin/disable/enable/delete sends the cached version as expected_version; a 409 memory_conflict reuses handle409Conflict (amber toast + auto-refetch GET item + re-enable retry); the new version replaces the cached one on success — no optimistic flip-then-revert (D-11) |
| T-02C2-02 | Operator error (destructive) | delete / disable without confirm | mitigate [BLOCKING] | Delete is gated behind the Phase-1 RED destructive dialog (repeats verb, states irreversibility, Cancel default-focused); disable behind a neutral confirm; pessimistic reflect-after-200 — the row is removed only after the backend confirms, so a mis-click or silent failure cannot hide a still-present item. Acceptance grep asserts the dialog copy |

| T-02C2-04 | Info disclosure | lifecycle failure toast | accept | SHELL-06 failure toast carries the real upstream status+message + Copy error; internal operator tool (BFF-04 passthrough) |
