<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| customer-support → console (rendered output) | Agent/answer/error strings (untrusted) are rendered into the DOM here. |
| console → BFF (request) | Chat is auth-none; the page must send no auth/scope/session header (the client from 04-01 enforces this). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering / Elevation | MessageBubble / ChatPage rendering | mitigate | Render EVERY message/answer/error string as a React TEXT node (children, not innerHTML); no dangerouslySetInnerHTML. React escapes by default; agent output is untrusted. (V7 Output Encoding) |
| T-04-05 | Spoofing | session_id display | accept | session_id shown via CopyableId is opaque continuity, not a credential (auth-none); displaying/copying it grants no privilege. Not persisted to localStorage (D-06 ephemeral). |
| T-04-06 | Information Disclosure | conversation content | mitigate | The console does not log message/answer bodies to any sink; raw frames are surfaced only in the operator UI (RawJsonViewer, added in 04-03), never shipped/persisted. (mirrors Phase-1 body pass-through posture) |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | None — this plan installs NO package. No install task → no legitimacy checkpoint needed. |
</threat_model>
