<verified_contract>
<!-- All [VERIFIED] from customer-support Go source — see RESEARCH "The Verified Chat Contract". -->
- BFF mount: CHAT_BASE='/api/chat'; StripPrefix removes it. Stream URL = `${CHAT_BASE}/chat/stream` → upstream POST /chat/stream. Sync URL = `${CHAT_BASE}/chat` → upstream POST /chat.
- Request DTO (both): { message: string; session_id?: string }. session_id OMITTED on turn 1; prior id echoed on every later turn — in the BODY, NEVER a header.
- Sync response DTO: { answer: string; agent: string; session_id?: string }.
- Stream frames: SSE named events `step` | `done` | `error`, each data = StreamEnvelope { kind: string; answer?: string; error?: string }.
    step  → kind=agent StepKind (thought·action·observation·reflection·plan·final); text is in `answer` (NOT content); NOT terminal.
    done  → kind="done"; final answer in `answer` (may be EMPTY — Pitfall 5 bare-done); TERMINAL.
    error → kind="error"; failure string in `error`; TERMINAL.
  Exactly ONE terminal frame, then the stream closes. `kind` is an OPEN string → parse loosely; unknown kind → keep as a neutral step row, never crash.
- Step.Tool/Args/Result are NOT on the wire — only kind + answer. Do NOT model tool/args fields.
- Session: server sets `X-Session-Id` on the open response BEFORE WriteHeader(200) (stream path, readable by onOpen) AND in the sync JSON body. Console reads it, echoes in the next request body session_id.
- Errors are FLAT { "error": "string" } — a chat-specific parseChatError; NOT Phase-2 parseGatewayError.
- Auth: NONE. Send ONLY Content-Type: application/json + body. No Authorization, no X-Console-*, no X-Session-Id header.
</verified_contract>
