<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| customer-support → console (streamed frames) | Untrusted step/answer/error strings and raw frame payloads are rendered into the trace/bubble here. |
| operator → in-flight stream (Stop) | An operator-initiated abort must read as benign (closed), never as a failure (errored) — a UX-integrity boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Tampering / Elevation | StepTrace / MessageBubble / ChatPage rendering | mitigate | Render every step text, answer, and error string as a React TEXT node; the per-step raw frame goes through RawJsonViewer (text). No dangerouslySetInnerHTML. Agent/tool output is untrusted. (V7 Output Encoding) |
| T-04-08 | Tampering | unknown/malformed step kind | mitigate | StepTrace falls back to a neutral `circle` icon for an unknown `kind` (open string) and never crashes; the 04-01 hook already drops malformed frames. (V5 Input Validation) |
| T-04-09 | Repudiation / UX-integrity | Stop vs error vs drop signaling | mitigate | The connection machine (04-01) maps Stop→terminal→closed (never errored); the UI renders three DISTINCT markers (muted Stopped / red Failed / amber Connection lost) so an operator can never misread "I stopped this" as a failure (D-05). |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | None — this plan installs NO package (the Stream|Sync toggle reuses the installed `button` block, NOT a new `switch`). No install task → no legitimacy checkpoint. |
</threat_model>
