<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → BFF `/api/health` | client reads the aggregate health DTO; must never receive internal topology |
| BFF → 3 upstreams | server-side probes to fixed config URLs (no client-supplied target) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-leak | Information Disclosure | `/api/health` response DTO | mitigate | DTO carries ONLY status/lastChecked/latencyMs — never the upstream URL or err.Error() (Pitfall 3). Go test asserts no upstream host string and no error/url field appears in the body. **Block-worthy.** |
| T-05-ssrf | Tampering / Info Disclosure | `/api/health` probe targets | mitigate | Probe URLs are FIXED from cfg.{Flow,Chat,Memory}Base; the handler accepts no path/query input — not an open proxy. |
| T-05-metrics | Information Disclosure / DoS | memory `/metrics` body | mitigate | Status-code-only probe; io.Copy(io.Discard, body); the Prometheus body is never parsed nor forwarded to the SPA. |
| T-05-secret | Information Disclosure | cfg.FlowdToken / OperatorToken | mitigate | Health endpoints are auth-none; no secret participates in any probe and none appears in the DTO. |
| T-05-xss | Tampering (XSS) | health/last-checked strings in TopBar | mitigate | All strings render as TEXT nodes (inherited Phase 2-4 posture); no dangerouslySetInnerHTML. |

Probe posture: all three upstream health endpoints are auth-none by design (verified) — no new auth surface; `/api/health` sits inside the existing MiddlewareOperatorAuth wrap (V2 partial).
