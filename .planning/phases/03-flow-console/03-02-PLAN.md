<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| console → BFF `/api/flow/*` | CRUD calls carry NO bearer / NO X-Console-* (BFF injects/strips) |
| flowd flow JSON + name → DOM | Flow `name`, flow IR text, and error strings rendered into the UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-V5 | Tampering / Elevation | FlowEditor + error/name rendering | mitigate | render ALL flowd strings (flow name, decoded flow JSON, error messages) as React TEXT nodes — never dangerouslySetInnerHTML (RESEARCH V5); the editor textarea holds plain text |
| T-03-04 | Tampering | PUT/POST flow body | mitigate | omit `id` from PUT body (or match URL id) + send raw flow, not base64 (Pitfall 4) — verified by FlowEditor.test asserting the request body |
| T-03-05 | Info disclosure | flowd error surfacing | accept | flat `{error:string}` is surfaced verbatim (BFF-04 passthrough); flowd's flat envelope carries no code/request_id/internal detail to leak |
| T-03-SC | Tampering | npm installs | mitigate | only the official shadcn `tabs` copy-in block is added (no npm runtime dep, official registry — RESEARCH Registry Safety, no vetting gate); no `npm install` |
</threat_model>
