## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| console → BFF `/api/flow/runs/*` + `/flows/{id}/runs` | NO bearer / NO X-Console-* (BFF injects); flowd single-tenant — all operators see all runs |
| replayed/persisted + live event payloads → DOM | Untrusted persisted/streamed event strings rendered into the sub-route timeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-V5 | Tampering / Elevation | RunDetail + live/replay TimelineView | mitigate | replayed + live payloads (persisted/streamed node outputs/errors) rendered as React TEXT nodes via the same TimelineView/RawJsonViewer — never dangerouslySetInnerHTML (RESEARCH V5) |
| T-03-07 | Tampering | replay/live-resume de-dup | mitigate | both replay and the mid-run /events hydrate feed the (kind,node,ordinal)-de-duping reducer (unit-tested in Plan 03) so history hydration + a deep-link mid-run can never double events (Pitfall 2) |
| T-03-10 | Info disclosure | cross-run event access | accept | flowd is single-tenant — all operators see all runs (internal ops tool; RBAC out of scope per REQUIREMENTS; auth lives at the BFF/ingress) |
| T-03-11 | Availability/correctness | empty /events handling | mitigate | 200+empty events → empty state, NOT an error (Pitfall 7) — asserted in RunDetail.test |
| T-03-SC | Tampering | npm installs | mitigate | no installs (RESEARCH); reuses existing components/primitives |
</threat_model>
