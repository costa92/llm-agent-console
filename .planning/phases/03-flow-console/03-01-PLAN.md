<flowd_contract>
<!-- Verified from flowd source (03-RESEARCH.md). The console calls /api/flow/* → BFF StripPrefix → flowd root. -->
REST routes (relative to /api/flow):
  GET  /flows                     → {flows:[FlowMeta]}
  POST /flows                     → 201 FlowRecord | 409 {error}
  GET  /flows/{id}                → FlowRecord | 404 {error}
  PUT  /flows/{id}                → 200 FlowRecord | 400 {error}
  DELETE /flows/{id}              → 204 No Content | 404 {error}   (NO body)
  POST /flows/{id}/run            → 200 {outputs, run_id} | 4xx/5xx {error}   (X-Run-ID header always set)
  GET  /flows/{id}/runs           → {runs:[RunMeta]}
  GET  /runs/{id}                 → RunRecord | 404 {error}
  GET  /runs/{id}/events          → 200 {events:[RunEvent]}  (200+empty for unknown/empty run; may 404)
SSE routes (POST):
  POST /flows/{id}/run/stream     → text/event-stream   (X-Run-ID set BEFORE the first frame)
  POST /runs/{id}/replay          → text/event-stream   (X-Run-ID + X-Replay:true headers)
DTOs:
  FlowMeta   = { id, name?, created_at, updated_at }
  FlowRecord = FlowMeta + { json: string }   // base64 of the flow IR bytes
  RunMeta    = { id, flow_id, status: 'running'|'done'|'failed', started_at, finished_at? }
  RunRecord  = RunMeta + { inputs?, outputs?, error? }
  RunEvent   = { seq, kind, node_id?, payload, ts }   // seq lives ONLY here, not in SSE frames
  SSE frame  = event:<kind>  data:<streamPayload-json>   (NO id/retry/heartbeat lines)
  6 kinds    = flow_started | node_started | node_finished | node_skipped | flow_done(terminal) | flow_err(terminal)
  streamPayload = only-populated-keys subset of { flow, node, input, output, outputs, error, metadata }
  flowd error envelope = { "error": "string" }   // FLAT — not the gateway {error:{code,...}}
Run inputs are string→string only: { inputs: { "<name>": "<string>" } }.
X-Run-ID: set on /run, /run/stream, and /replay responses — the live run's identity for the deep-linkable sub-route (D-08).
</flowd_contract>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| console (browser) → BFF `/api/flow/*` | Console sends NO bearer / NO X-Console-* on flow calls; the BFF flow director (Phase-1) strips inbound auth + X-Console-* and injects the flowd bearer server-side |
| flowd response → console state | Untrusted upstream JSON (incl. base64 flow IR + event payload strings + the X-Run-ID header) crosses into UI state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | flow client `/api/flow/*` calls | mitigate | client/stream send NO Authorization, NO X-Console-* (RESEARCH V4) — verified by code review + a client test asserting no auth headers are set |
| T-03-02 | Tampering | upstream flowd JSON → zod | mitigate | all REST 2xx bodies pass through zod schemas before reaching state (V5); flowdErrorSchema narrows the error body |
| T-03-03 | Info disclosure | flowd token via SSE response headers | accept (handled upstream) | Phase-1 ModifyResponse scrubs Authorization/X-Echo-Auth on flow responses; the console reads only `X-Run-ID` (a run identifier, not a secret) from the open Response and adds no client bearer |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs this phase (RESEARCH Package Legitimacy Audit: not applicable — `@microsoft/fetch-event-source` already installed). No `npm install` runs. |
