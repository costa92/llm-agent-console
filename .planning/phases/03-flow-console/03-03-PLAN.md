  <name>Pure timeline reducer + de-dup + per-node status</name>
  <files>web/src/features/flow/timeline/reducer.ts, web/src/features/flow/timeline/reducer.test.ts</files>
  <behavior>
    Golden success sequence (flow_started → node_started(A) → node_finished(A) → flow_done) → events appended in order; nodeStatus A=done; terminal='done'; outputs captured from flow_done.
    Failure sequence ending in flow_err → terminal='error'; the error payload captured; the attributable node (if node present) → errored; partial events retained.
    node_skipped variant → that node's status = skipped.
    De-dup: dispatch history events tagged source:'history' with seq [1,2,3], then live source:'live' [3,4,5] → merged log [1,2,3,4,5], the seq-3 overlap appears ONCE.
    Replay parity: feeding the SAME golden frames as source:'history' yields an identical render model to source:'live' (success criterion 5).
    reset action clears to the initial timeline.
  </behavior>
  <implementation>Write `timelineReducer(state, action)` as a PURE function (no I/O, no React). Action union: `{type:'reset'}` and `{type:'event', source:'live'|'history', kind:SseKind, payload:SsePayload}` (ordinal derived inside the reducer: for history use payload-carried seq if present else append-position; for live use append-position). State: `{ events: TimelineEvent[], seen: Set<string>, nodeStatus: Record<string,NodeStatus>, terminal: 'none'|'done'|'error', outputs?, error? }`. Compute the de-dup key `${kind}:${node ?? ''}:${ordinal}`; skip if already in `seen`. applyNodeStatus maps node_started→running, node_finished→done, node_skipped→skipped, and on flow_err mark the attributable node errored. Set terminal on flow_done/flow_err and capture outputs/error. Keep it immutable (new arrays/Sets). Export the initial state + types (TimelineEvent, NodeStatus, Timeline). Tests cover every behavior row above using the 03-01 golden fixtures.</implementation>
</feature>

<feature>
  <name>Connection-state machine</name>
  <files>web/src/features/flow/timeline/connection.ts, web/src/features/flow/timeline/connection.test.ts</files>
  <behavior>
    Initial state 'idle'; start → 'streaming'.
    flow_done or flow_err (terminal frame) → 'closed' (neutral; success/failure is the terminal frame's job, not the connection's).
    onError BEFORE any terminal frame → 'errored' ('Connection lost').
    A transition from a terminal 'closed' is a no-op (already ended); the union is extensible (a future 'reconnecting' slots between streaming and errored without rewrites).
  </behavior>
  <implementation>Define `type ConnState = 'idle'|'streaming'|'closed'|'errored'` (extensible) and a small pure transition function `connReducer(state, event)` over events `{type:'start'|'terminal'|'transport-error'|'reset'}`. terminal → closed; transport-error (only if not already closed) → errored; start → streaming; reset → idle. No booleans — the union is the source of truth so Phase 5 can insert 'reconnecting'. Unit-test each transition incl. the "terminal then late transport-error is ignored" guard.</implementation>
</feature>

<feature>
  <name>Imperative useRunStream hook</name>
  <files>web/src/features/flow/timeline/useRunStream.ts, web/src/features/flow/timeline/useRunStream.test.ts</files>
  <behavior>
    start(flowId, inputs) → opens runStream, sets connection 'streaming', resets the reducer, dispatches each frame into the reducer in arrival order.
    On a terminal frame (flow_done/flow_err) → connection 'closed' + AbortController.abort() (no leaked fetch — Pitfall 6).
    On transport onError before terminal → connection 'errored'.
    retry() re-opens the stream (manual Retry — D-09; Phase 5 automates).
    replay(runId) opens replayStream feeding the SAME reducer with source:'history'.
    Unmount → abort the in-flight stream.
  </behavior>
  <implementation>Write `useRunStream()` returning `{ timeline, conn, start, replay, retry }`. Internally `useReducer(timelineReducer)` + the connection machine (useState/useReducer over ConnState) + an `abortRef`. `start` creates an AbortController, dispatches reset, sets streaming, calls 03-01 `runStream` with onMessage → `JSON.parse(data)` → dispatch `{type:'event', source:'live', kind:event, payload}`; on flow_done/flow_err → conn 'closed' + abort; onError → conn 'errored'. `replay` mirrors start but uses `replayStream` + source:'history' (no auto-scroll concern here — that's the view). `useEffect` cleanup aborts on unmount. Test with `vi.mock('@microsoft/fetch-event-source')` (the 03-01 fake emitter): assert in-order dispatch, terminal flips conn + aborts, transport error → errored, unmount aborts. Mock the wrapper, not the network.</implementation>
