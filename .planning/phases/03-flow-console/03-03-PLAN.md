<feature>
  <name>Connection-state machine</name>
  <files>web/src/features/flow/timeline/connection.ts, web/src/features/flow/timeline/connection.test.ts</files>
  <behavior>
    Initial state 'idle'; start → 'streaming'.
    flow_done or flow_err (terminal frame) → 'closed' (neutral; success/failure is the terminal frame's job, not the connection's).
    onError BEFORE any terminal frame → 'errored' ('Connection lost').
    A transition from a terminal 'closed' is a no-op (already ended); the union is extensible (a future 'reconnecting' slots between streaming and errored without rewrites).
  </behavior>
  <implementation>Define `type ConnState = 'idle'|'streaming'|'closed'|'errored'` (extensible) and a small pure transition function `connReducer(state, event)` over events `{type:'start'|'terminal'|'transport-error'|'reset'}`. terminal → closed; transport-error (only if not already closed) → errored; start → streaming; reset → idle. No booleans — the union is the source of truth so Phase 5 can insert 'reconnecting'. Unit-test each transition incl. the "terminal then late transport-error is ignored" guard.</implementation>
</feature>

| Boundary | Description |
|----------|-------------|
| SSE frame payload → reducer state | Untrusted streamed event payloads (LLM/tool output strings) enter the render model |
| browser fetch lifecycle → flowd run | An un-aborted stream leaves a flowd run executing detached |
| X-Run-ID open header → navigation | A run identifier from the upstream response drives a client-side route change |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | DoS / resource | useRunStream AbortController | mitigate | terminal frame AND unmount both abort the fetch → flowd's r.Context() cancels → engine stops (RESEARCH "Detached/leaked run"); verified by the unmount-aborts + terminal-aborts tests |
| T-03-V5 | Tampering | reducer payload handling | mitigate | the reducer stores payloads as data only (no HTML/eval); rendering-as-text is enforced in Plan 04's TimelineView. The reducer never interprets strings as markup |
| T-03-07 | Tampering | de-dup correctness | mitigate | (kind,node,ordinal) de-dup unit-tested with the history/live overlap fixture so replay/late-join + the transport-drop /events hydrate can never silently double events (Pitfall 2) |
